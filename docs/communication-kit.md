# POP Communication Kit

This document standardizes external messaging for Persona Object Protocol (POP). It is positioning-first and non-normative. The protocol definition remains in [`spec/pop-0.1.md`](../spec/pop-0.1.md).

For platform-specific short copy, see [`short-outreach-kit.md`](./short-outreach-kit.md).

## Core line

English:

POP (Persona Object Protocol) is a lightweight public draft protocol for portable persona objects in multimodal AI systems.

中文：

POP（Persona Object Protocol）是面向多模态 AI 系统的可移植人格对象轻量公开协议草案。

Use this line by default. Avoid swapping in alternative headline definitions unless there is a specific reason.

## Expanded introduction

### Version A

POP does not define persona generation algorithms. It defines how persona objects can be represented, transferred, and attached to capabilities.

It emphasizes a basic separation between persona, memory, permissions, and tool capability.

Its goal is to provide a minimal persona object layer for future multimodal AI systems.

### Version B

POP is not primarily about what interface an AI uses. It is about how the same persona can be expressed in a stable way, carried across substrates, and bounded by basic governance constraints.

It does not compete for the application layer. It positions itself at the persona object layer.

Its aim is to define a portable persona entry point for future multimodal assistants, agents, and companion systems.

我发布了 POP（Persona Object Protocol）。

这是一个面向多模态 AI 系统的可移植人格对象轻量公开协议草案，用来定义人格对象如何被表示、迁移和挂接能力。

仓库与 DOI 已公开。

### Standard

我刚公开了一个新草案项目：POP（Persona Object Protocol）。

它不是一个陪伴 AI 产品，也不是人格生成算法，而是一个面向多模态 AI 系统的可移植人格对象协议草案。

它主要解决的是：当 AI 人格已经存在时，能不能把它表达成一个相对清晰、可迁移、可挂接能力、并与记忆和权限分离的对象。

当前版本已经完成：

- GitHub 仓库公开
- v0.1 draft release
- Zenodo DOI 归档

### Positioning-first

我最近完成了一个新方向的第一轮卡位：POP（Persona Object Protocol）。

我的判断是，未来多模态 AI 世界里，真正值得提前占位的，不只是工具协议、agent 协议和记忆系统，还有一个更基础的问题：人格能不能成为对象。

POP 的位置，不在应用层，而在“人格对象层”。

它不定义人格怎么生成，而是定义人格如何被表示、迁移、挂接能力，并与记忆、权限和工具边界分开。

现在这个项目已经完成公开仓库、release 和 DOI 归档，先把这个入口占住。

## Standard responses

### What is this?

这是一个人格对象协议草案，不是应用产品。

### How does this relate to MCP?

MCP 解决工具连接，POP 解决人格对象表示。两者互补，不冲突。

### Are you building virtual romance products?

不是。POP 不定义具体应用场景，它定义的是更底层的人格对象层。

### Are you building persona generation algorithms?

不是。POP 不做人格生成方法，它做的是人格对象的表达与边界。

## Avoid as primary framing

Do not use these as the main public framing at this stage:

- digital immortality
- perfect companion robots
- deceased-person revival
- ideal lover protocol
- cross-model high-fidelity persona injection

These may distract from the protocol layer and pull discussion into application controversy.
