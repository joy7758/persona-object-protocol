# POP Short Outreach Kit

This document provides short-form public messaging for POP across common channels. It complements [`communication-kit.md`](./communication-kit.md) and keeps the framing compact, repeatable, and positioning-first.

## Default recommendation

Use the positioning-first version by default when announcing POP publicly:

我最近完成了一个新入口的第一轮卡位：**POP（Persona Object Protocol）**。

我的判断是，未来多模态 AI 世界里，不只是工具和 agent 需要协议，**人格对象层**也需要最小表达标准。

POP 先占这个位置：定义人格对象如何被表示、迁移，并与记忆、权限和工具边界分开。

仓库、Release 和 DOI 已公开。

## Reusable short lines

Use these as interchangeable second or third lines:

- 这是面向多模态 AI 系统的可移植人格对象协议草案。
- 它不定义人格生成算法，而是定义人格对象如何被表示、迁移和挂接能力。
- 它强调人格、记忆、权限与工具能力之间的基础分离。
- 它的位置不在应用层，而在“人格对象层”。
- 这不是产品页，而是一个协议入口。

## Use notes

- Default to the positioning-first version unless the audience only needs a plain definition.
- Avoid dragging the message into romance, deceased-person, or digital immortality narratives.
- Keep one core sentence stable across platforms to reduce concept drift.
